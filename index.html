Pengenalan JavaScript

JavaScript (JS) adalah bahasa pemrograman yang digunakan untuk membuat website menjadi interaktif, dinamis, dan hidup.

1. Peran JavaScript di Web

Jika kita mengibaratkan website sebagai manusia, maka:

  • HTML (Structure): Adalah Tulang/Kerangka (Menentukan isi dan struktur).
  • CSS (Style): Adalah Baju/Penampilan (Mengatur warna, font, dan tata letak).
  • JavaScript (Behavior): Adalah Otak/Otot (Menangani logika, klik tombol, animasi, dan data).
Fakta Penting: JavaScript TIDAK SAMA dengan Java. Keduanya adalah bahasa yang sangat berbeda, layaknya "Car" (Mobil) dan "Carpet" (Karpet).

2. Cara Menulis JavaScript

Kode JS dapat ditulis langsung di dalam file HTML menggunakan tag <script>, atau di file terpisah (External).

a. Internal (Di dalam HTML)

Biasanya diletakkan tepat sebelum penutup tag </body> agar website loading lebih cepat.

HTML / CSS
<!DOCTYPE html><html><body> <h1>Halo Dunia</h1> <!-- Kode JS ditulis di sini --> <script> alert("Selamat Datang!"); </script></body></html>

b. External (File Terpisah - Best Practice)

Digunakan untuk proyek profesional agar kode rapi.

HTML / CSS
<!-- Menghubungkan ke file script.js --><script src="script.js"></script>

3. Cara Menampilkan Output

Sebagai developer, kita sering menggunakan console untuk mengecek apakah kode berjalan benar.

HTML / CSS
console.log("Pesan ini muncul di Console browser");alert("Ini adalah notifikasi!");document.write("Teks ini ditulis oleh JavaScript");

4. Kenapa JavaScript Sangat Populer?

Saat ini JS adalah bahasa pemrograman paling populer di dunia karena:

  • Fullstack: Bisa jalan di Browser (Frontend) dan Server (Backend dengan Node.js).
  • Framework Besar: Memiliki ekosistem raksasa seperti React, Vue, dan Angular.
  • Komunitas: Sangat mudah mencari solusi jika terjadi error.

Variables & Data Types

Variabel adalah wadah untuk menyimpan data. Bayangkan variabel seperti sebuah kardus yang kita beri label nama agar isinya bisa kita cari dan gunakan nanti.

1. Deklarasi Variabel (Var, Let, Const)

Di JavaScript modern, ada tiga cara membuat variabel. Namun, cara penggunaannya berbeda.

a. var (Cara Lama / Legacy)

Dulu ini satu-satunya cara. Sekarang tidak disarankan karena memiliki masalah pada lingkup (scope) yang bisa menyebabkan bug.

HTML / CSS
var nama = "Budi";var nama = "Andi"; console.log(nama);
Hindari var! Gunakan hanya jika kamu memelihara kode legacy (kode tua). Untuk proyek baru, selalu gunakan let atau const.

b. let (Nilai Bisa Berubah)

Gunakan let jika nilai variabel tersebut akan berubah di masa depan (misalnya: skor game, counter, umur).

HTML / CSS
let skor = 0;skor = 10; skor = 20;

c. const (Nilai Tetap / Konstanta)

Gunakan const untuk nilai yang tidak boleh berubah selamanya (misalnya: ID User, Phi, Rumus pasti). Ini adalah default terbaik.

HTML / CSS
const phi = 3.14;console.log(phi);
Tips Profesional: Selalu gunakan const secara default. Hanya ganti ke let jika kamu yakin nilainya perlu diubah nanti.

2. Tipe Data Dasar (JS Types)

JavaScript memiliki tipe data dinamis (Dynamic Typing), artinya kita tidak perlu menulis tipe datanya secara manual, komputer akan menebaknya sendiri.

a. String (Teks)

Kumpulan karakter yang diapit oleh tanda kutip ("", '', atau backticks ``).

HTML / CSS
let namaDepan = "John"; let namaBelakang = 'Doe'; let sapaan = `Halo ${namaDepan}`;

b. Number (Angka)

Mencakup bilangan bulat (integer) dan desimal (float).

HTML / CSS
let umur = 25;let harga = 15000.50;let suhu = -5;

c. Boolean (Logika)

Hanya memiliki dua nilai: true (Benar) atau false (Salah). Sering dipakai untuk pengkondisian (If/Else).

HTML / CSS
let isOnline = true;let isGameOver = false;

d. Empty Values (Null & Undefined)

Dua tipe data spesial yang sering membingungkan pemula.

  • Undefined: Variabel sudah dibuat, tapi belum diisi apa-apa. (Kosong tidak sengaja).
  • Null: Variabel sengaja dikosongkan oleh programmer. (Kosong sengaja).
HTML / CSS
let kotakKosong; console.log(kotakKosong); let kotakSampah = null;console.log(kotakSampah);

3. Cek Tipe Data

Gunakan operator typeof untuk mengetahui tipe data sebuah variabel.

HTML / CSS
console.log(typeof "Halo"); console.log(typeof 100); console.log(typeof true);

JavaScript Operators

Operator adalah simbol khusus yang digunakan untuk melakukan operasi pada nilai (operands). Tanpa operator, data hanya akan diam dan tidak bisa diolah.

1. Arithmetic Operators (Matematika)

Digunakan untuk melakukan perhitungan matematika standar.

a. Operasi Dasar

HTML / CSS
let x = 10;let y = 5;console.log(x + y); console.log(x - y); console.log(x * y); console.log(x / y);

b. Modulus (Sisa Bagi) & Eksponen

Modulus (%) sangat berguna untuk logika ganjil/genap atau batasan angka.

HTML / CSS
console.log(10 % 3); console.log(2 ** 3);

c. Increment & Decrement (++, --)

Menambah atau mengurangi nilai sebesar 1. Hati-hati penempatannya!

  • variable++ (Postfix): Gunakan nilai dulu, baru tambah.
  • ++variable (Prefix): Tambah dulu, baru gunakan nilai.
HTML / CSS
let angka = 5;console.log(angka++); console.log(angka); let nilai = 5;console.log(++nilai);

2. Assignment Operators (Pemberian Nilai)

Digunakan untuk memasukkan nilai ke dalam variabel. Simbol dasarnya adalah =.

a. Compound Assignment (Singkatan)

Cara cepat melakukan operasi lalu menyimpannya kembali.

HTML / CSS
let saldo = 10000;saldo += 5000; saldo -= 2000; saldo *= 2;

b. Logical Assignment (ES2021 Modern)

Fitur canggih untuk menetapkan nilai default jika datanya kosong.

  • ||= : Isi jika nilainya Falsy.
  • ??= (Nullish): Isi HANYA jika nilainya null atau undefined (Lebih aman).
HTML / CSS
let namaUser = null;namaUser ??= "Tamu"; console.log(namaUser);

c. Simbol Titik Dua (Colon :)

Simbol :bukan operator assignment, tapi sering muncul berdampingan. Digunakan dalam:

  • Object Property: Pemisah antara Key dan Value (nama: "Budi").
  • Ternary Operator: Sebagai pemisah kondisi Else.
HTML / CSS
let status = (umur >= 17) ? "Dewasa" : "Anak-anak";

3. Comparison Operators (Perbandingan)

Digunakan untuk membandingkan dua nilai. Hasilnya SELALU Boolean (true atau false).

a. Relational

HTML / CSS
console.log(10 > 5); console.log(10 < 5); console.log(10 >= 10);

b. Equality (Sama Dengan) - HATI-HATI!

Ini adalah sumber bug terbesar bagi pemula. Ada dua jenis persamaan:

Perbedaan Fatal:
  • == (Loose): Hanya cek nilai (Bisa menipu).
  • === (Strict): Cek Nilai DAN Tipe Data (Wajib dipakai).
HTML / CSS
let angka = 5; let teks = "5"; console.log(angka == teks); console.log(angka === teks); console.log(angka !== teks);
Best Practice: Selalu gunakan === dan !== untuk menghindari error yang sulit dilacak. Perlakukan tipe data yang berbeda sebagai entitas yang berbeda.

Control Flow & Conditionals

Secara default, kode JavaScript berjalan lurus dari atas ke bawah. Control Flow memungkinkan kita mengontrol alur tersebut: membelokkan, melompati, atau mengulang kode berdasarkan kondisi tertentu.

1. Booleans & Logical Logic (Pondasi)

Sebelum membuat keputusan (If), kita harus paham dulu tentang nilai Kebenaran (True/False).

a. Truthy & Falsy Values

Ini adalah konsep unik di JavaScript. Semua nilai dianggap True (Truthy), KECUALI 6 nilai berikut yang dianggap False (Falsy):

6 Nilai Falsy (Hafalkan ini!):
  1. false
  2. 0 (Angka nol)
  3. "" (String kosong)
  4. null
  5. undefined
  6. NaN (Not a Number)
HTML / CSS
let inputUser = ""; if (inputUser) { console.log("User mengisi data");} else { console.log("Data kosong!"); }

b. Logical Operators (&&, ||, !)

Digunakan untuk menggabungkan logika keputusan.

  • && (AND): Semua kondisi harus Benar.
  • || (OR): Salah satu kondisi saja Benar, maka hasil Benar.
  • ! (NOT): Membalikkan nilai (True jadi False).
HTML / CSS
const umur = 20;const punyaSIM = true;const isMabuk = false;if (umur >= 17 && punyaSIM && !isMabuk) { console.log("Boleh menyetir mobil");} else { console.log("Dilarang menyetir!");}

2. If, Else, & Else If

Struktur paling dasar untuk percabangan logika.

a. Syntax Dasar

HTML / CSS
let nilai = 75;if (nilai >= 90) { console.log("Grade A");} else if (nilai >= 70) { console.log("Grade B"); } else { console.log("Remedial");}

b. Nested If (If Bersarang)

Kita bisa menaruh If di dalam If. Namun, hati-hati! Terlalu banyak sarang membuat kode sulit dibaca (Spaghetti Code).

HTML / CSS
const password = "rahasia";if (password.length >= 5) { if (password.indexOf(" ") === -1) { console.log("Password Valid (Panjang & Tanpa Spasi)"); } else { console.log("Password tidak boleh ada spasi!"); }} else { console.log("Password terlalu pendek!");}

3. Best Practice: Guard Clauses

Hindari Nested If (If bersarang) yang terlalu dalam karena sulit dibaca. Gunakan teknik Guard Clauses (Cek error dulu, lalu return).

Pro Tip: Usir kondisi yang salah di awal fungsi agar kode utama tetap bersih.
HTML / CSS
function login(user) { if (user) { if (user.password === "123") { return "Login Sukses"; } }}function loginPro(user) { if (!user) return "User tidak ditemukan"; if (user.password !== "123") return "Password Salah"; return "Login Sukses";}

3. Switch Statement

Gunakan switch jika kamu ingin membandingkan satu variabel dengan banyak nilai spesifik. Ini lebih rapi daripada menggunakan banyak else if.

Penting: Jangan lupa kata kunci break! Jika lupa, komputer akan terus menjalankan case di bawahnya (disebut "Fall-through").
HTML / CSS
const hari = 2; switch (hari) { case 1: console.log("Senin: Upacara"); break; case 2: console.log("Selasa: Olahraga"); break; case 3: console.log("Rabu: Produktif"); break; case 4: case 5: console.log("Kamis/Jumat: Semangat!"); break; default: console.log("Hari Libur / Tidak Valid");}

4. Ternary Operator (Shorthand)

Cara singkat menulis if...else dalam satu baris. Sangat sering digunakan di framework modern seperti React.

Rumus:kondisi ? "Jika Benar" : "Jika Salah"

HTML / CSS
let statusMember = "Gold";let diskon;if (statusMember === "Gold") { diskon = 0.2;} else { diskon = 0;}let diskonBaru = (statusMember === "Gold") ? 0.2 : 0;console.log(`Diskon Anda: ${diskonBaru * 100}%`);
Best Practice Summary:
  • Gunakan Ternary untuk kondisi sederhana (If-Else pendek).
  • Gunakan Switch untuk mengecek banyak nilai pasti.
  • Gunakan If/Else untuk logika yang kompleks atau pengecekan range (lebih besar/kecil).

JavaScript Loops (Perulangan)

Loops memungkinkan kita menjalankan blok kode yang sama berulang kali tanpa harus menulisnya berkali-kali. Ini adalah penerapan prinsip DRY (Don't Repeat Yourself).

1. The For Loop

Digunakan ketika kita tahu pasti berapa kali kita ingin melakukan perulangan. Ini adalah jenis loop yang paling sering digunakan.

Rumus:for (mulai; kondisi; langkah) { ... }
HTML / CSS
for (let i = 1; i <= 5; i++) { console.log("Putaran ke-" + i);}

Mengakses Array dengan Loop

Kekuatan utama loop adalah untuk memproses data list (Array).

HTML / CSS
const buah = ["Apel", "Jeruk", "Mangga"];for (let i = 0; i < buah.length; i++) { console.log(buah[i]); }

2. The While Loop

Digunakan ketika kita TIDAK tahu berapa kali harus mengulang, tapi kita tahu kondisi berhentinya.

Analogi: "Makan terus (Loop) SELAMA piring masih ada isinya (Condition)."
HTML / CSS
let nyawaPlayer = 5;while (nyawaPlayer > 0) { console.log("Player masih hidup! Nyawa: " + nyawaPlayer); nyawaPlayer--; }console.log("Game Over");
Bahaya: Infinite Loop! Jika kamu lupa menulis nyawaPlayer-- (penghancur kondisi), loop tidak akan pernah berhenti dan akan membuat browser/komputer hang (crash).

3. Break Statement

break digunakan untuk menghentikan paksa perulangan, meskipun kondisinya masih True.

Biasanya dipakai saat mencari data: "Kalau data sudah ketemu, stop mencari!"

HTML / CSS
for (let i = 1; i <= 10; i++) { if (i === 5) { console.log("Ditemukan angka 5, berhenti!"); break; } console.log(i);}

4. Continue Statement

continue digunakan untuk melewati (skip) satu putaran saat ini, dan langsung loncat ke putaran berikutnya.

Biasanya dipakai untuk memfilter data.

HTML / CSS
for (let i = 0; i <= 5; i++) { if (i % 2 === 0) { continue; } console.log("Angka Ganjil: " + i);}
Ringkasan:
  • Break: "Saya menyerah, keluar dari sini." (Stop Total)
  • Continue: "Saya skip bagian ini, lanjut ke yang berikutnya." (Skip Satu)

Real World Logic: Diskon & Transaksi

Kumpulan pola logika (Design Patterns) yang paling sering digunakan dalam pembuatan aplikasi toko online, kasir, dan keuangan.

1. Rumus Matematika Diskon

Banyak pemula salah dalam menghitung persen. Berikut adalah cara penulisan yang benar di JavaScript.

a. Menghitung Potongan Harga (Nominal)

Rumus:Harga Awal * (Persen / 100)
HTML / CSS
let harga = 100000;let diskon = 20; let potongan = harga * 0.2; let potonganLain = harga * (diskon / 100); console.log(potongan);

b. Update Harga Bertingkat (Accumulator Pattern)

Saat menghitung diskon bertumpuk (Diskon Qty -> Diskon Member -> Voucher), jangan buat variabel baru untuk setiap harga. Update variabel yang sama.

HTML / CSS
let totalBayar = 1000000;totalBayar -= (totalBayar * 0.1); totalBayar -= 50000;

2. Diskon Kuantitas (Tiered Logic)

Gunakan if-else untuk mengecek jumlah pembelian. Tips: Cek dari jumlah terbesar dulu!

HTML / CSS
let qty = 6;let diskonPersen = 0;if (qty >= 10) { diskonPersen = 0.50; } else if (qty >= 5) { diskonPersen = 0.20; } else { diskonPersen = 0; }

3. Logika Member (Switch Case)

Gunakan switch ketika opsinya sudah pasti (Fixed values).

HTML / CSS
let member = "VIP";let potongan = 0;switch (member) { case "VIP": potongan = 100000; break; case "Reseller": potongan = 50000; break; default: potongan = 0; }

4. Logika Ongkir (Ternary Operator)

Gunakan logika ini untuk pengecekan sederhana (Ya/Tidak).

HTML / CSS
let lokasi = "Luar Jawa";let ongkir = (lokasi === "Jawa") ? 25000 : 50000;

5. Validasi Voucher (Complex Logic)

Menggunakan Operator Logika && (AND) untuk syarat ganda.

HTML / CSS
let kode = "HEMAT50";let totalBelanja = 40000;if (kode === "HEMAT50" && totalBelanja >= 50000) { console.log("Voucher Berhasil!");} else { console.log("Voucher Gagal: Min. belanja kurang atau kode salah.");}

6. Bonus: Format Rupiah

Agar angka 15000000 berubah otomatis menjadi 15.000.000.

HTML / CSS
let uang = 15000000;console.log("Rp " + uang.toLocaleString('id-ID'));

JavaScript Functions

Function adalah blok kode yang dirancang untuk melakukan tugas tertentu. Kita bisa membayangkannya sebagai "Mesin" atau "Rumus" yang bisa kita panggil kapan saja.

1. Analogi & Syntax Dasar

Analogi Blender:
  • Input (Parameter): Buah, Air, Es Batu.
  • Proses (Function): Pisau berputar menghancurkan bahan.
  • Output (Return): Jus Buah.
JavaScript
function sapaDunia() { console.log("Halo Dunia!"); console.log("Saya sedang belajar JS Function");}sapaDunia(); sapaDunia();

2. Parameter & Argument (Input)

Agar function dinamis, kita bisa mengirimkan data ke dalamnya.

  • Parameter: Variabel yang ditulis saat membuat fungsi (wadah).
  • Argument: Nilai asli yang dikirim saat memanggil fungsi (isi).
JavaScript
function sapaUser(nama) { console.log("Selamat Pagi, " + nama + "!");}sapaUser("Budi"); sapaUser("Siti");

3. The Return Statement (Output)

Ini konsep yang sering disalahpahami pemula. console.log hanya menampilkan, tapi returnmengembalikan nilai agar bisa diolah lagi.

Aturan Return: Saat komputer bertemu kata kunci return, fungsi akan BERHENTI saat itu juga (kode di bawahnya tidak akan dijalankan).
JavaScript
function hitungLuas(panjang, lebar) { let hasil = panjang * lebar; return hasil; console.log("Ini tidak akan pernah muncul"); }let luasMeja = hitungLuas(10, 5); let luasLantai = hitungLuas(20, 20);console.log("Total Luas: " + (luasMeja + luasLantai));

4. Arrow Function (ES6 Modern)

Cara penulisan fungsi yang lebih singkat. Ini standar industri saat ini (terutama di React/Vue).

JavaScript
function tambah(a, b) { return a + b;}const tambahBaru = (a, b) => { return a + b;}const kali = (a, b) => a * b;console.log(kali(5, 5));

5. Professional Best Practices

  • Naming Convention: Gunakan kata kerja di depan.
    • ‚úÖ hitungTotal(), getUser(), checkLogin()
    • ‚ùå total(), user() (Ambigu)
  • Single Responsibility: Satu function sebaiknya hanya melakukan SATU tugas. Jangan gabungkan hitung pajak, kirim email, dan update database dalam satu function raksasa.

6. Studi Kasus: Sistem Kasir Restoran

Contoh penerapan Function untuk memecah logika program yang rumit menjadi bagian-bagian kecil (Modular Code).

Skenario

Kita akan membuat program kasir yang bisa:

  1. Menghitung harga makanan berdasarkan jumlah.
  2. Mengecek apakah pelanggan punya kartu member.
  3. Menghitung pajak restoran (10%).
  4. Mencetak struk total.

Kode Program (Full Version)

Perhatikan bagaimana kita membuat 3 function kecil (hitungDiskon, hitungPajak, cetakStruk) lalu menggabungkannya di function utama transaksiRestoran.

JavaScript
const hitungDiskon = (totalHarga, isMember) => { let diskon = 0; if (isMember === true) { if (totalHarga > 100000) { diskon = totalHarga * 0.20; } else { diskon = totalHarga * 0.10; } } return diskon; }const hitungPajak = (hargaBersih) => { const tarifPajak = 0.10; return hargaBersih * tarifPajak;}const formatRupiah = (angka) => { return "Rp " + angka.toLocaleString('id-ID');}function transaksiRestoran(namaPelanggan, totalBelanja, statusMember) { console.log("================================"); console.log(`PELANGGAN: ${namaPelanggan}`); console.log(`Total Awal: ${formatRupiah(totalBelanja)}`); console.log("--------------------------------"); const potongan = hitungDiskon(totalBelanja, statusMember); const hargaSetelahDiskon = totalBelanja - potongan; console.log(`Diskon : -${formatRupiah(potongan)}`); const pajak = hitungPajak(hargaSetelahDiskon); const grandTotal = hargaSetelahDiskon + pajak; console.log(`Pajak (10%): +${formatRupiah(pajak)}`); console.log("--------------------------------"); console.log(`TOTAL BAYAR: ${formatRupiah(grandTotal)}`); console.log("================================"); return grandTotal; }transaksiRestoran("Budi (Member)", 200000, true);transaksiRestoran("Asep (Tamu)", 50000, false);

Analisis Kode

  • Modularitas: Jika besok pajak naik jadi 11%, kita hanya perlu mengubah function hitungPajak. Function lain tidak akan rusak. Inilah keuntungan memecah kode.
  • Reusability: Function formatRupiah dipanggil 4 kali. Bayangkan kalau kita harus mengetik .toLocaleString() berulang-ulang, pasti capek.
  • Keterbacaan: Function utama transaksiRestoran sangat mudah dibaca alurnya karena detail rumitnya disembunyikan di function lain.

DOM Manipulation

Document Object Model (DOM) adalah representasi struktur HTML yang dapat dimanipulasi dengan JavaScript.

Materi tentang cara mengubah HTML menggunakan JavaScript akan ditambahkan di sini.
